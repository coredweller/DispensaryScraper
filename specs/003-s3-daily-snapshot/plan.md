# Implementation Plan: Daily Full-Menu Snapshot to S3

**Branch**: `003-s3-daily-snapshot` | **Date**: 2026-04-04 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/003-s3-daily-snapshot/spec.md`

## Summary

After each daily scrape completes, upload a structured JSON file containing every product from the full dispensary menu (not just filtered brands) to an S3 bucket under a date-keyed path (`runs/YYYY-MM-DD.json`). The snapshot includes run-level metadata and per-product numeric fields. The upload runs after scraping regardless of email outcome; failures are logged and do not crash the process or affect email delivery.

Technical approach: extend `Product` to `ScrapedProduct` (numeric fields), wrap `scrape()` return in `ScrapeOutput`, make `filter()` generic, derive email display values from numeric fields, and introduce `src/store.ts` for the S3 upload with 30s timeout and one-retry policy.

## Technical Context

**Language/Version**: TypeScript 5.5 / Node.js v18+
**Primary Dependencies**: puppeteer, nodemailer, dotenv, zod, @aws-sdk/client-s3
**Storage**: AWS S3
**Testing**: Manual E2E
**Target Platform**: Linux / GitHub Actions
**Project Type**: cli
**Performance Goals**: Upload within 30s per run
**Constraints**: Upload failure must not block email; no credentials in logs
**Scale/Scope**: ~50 products/day, one file/day
